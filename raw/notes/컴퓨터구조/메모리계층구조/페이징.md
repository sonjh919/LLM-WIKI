#컴퓨터구조

### 페이징이란

페이징(Paging)은 메모리를 **고정 크기의 블록**으로 나누어 관리하는 기법입니다. 가상 메모리는 **페이지(Page)**, 물리 메모리는 **프레임(Frame)**으로 나뉩니다.

### 페이징의 두 가지 역할

1. **메모리 할당 방식** - [[링크/컴퓨터구조/메모리계층구조/세그먼테이션]]의 외부 단편화 문제 해결
2. **[[링크/컴퓨터구조/메모리계층구조/가상메모리/가상 메모리]] 구현 수단** - 페이지 테이블을 통한 주소 변환

### 페이지와 프레임

**페이지**: 가상 메모리를 나눈 고정 크기 블록 (보통 4KB)
**프레임**: 물리 [[RAM]]을 나눈 고정 크기 블록 (페이지와 동일 크기)
**페이지 테이블**: 페이지 번호 → 프레임 번호 매핑 정보

### 주소 변환 과정

가상 주소는 **페이지 번호 + 오프셋**으로 구성됩니다. [[링크/컴퓨터구조/메모리계층구조/가상메모리/페이지 테이블]]에서 페이지 번호를 프레임 번호로 변환하고, 오프셋을 그대로 사용하여 물리 주소를 계산합니다.

```mermaid
flowchart LR
    VA[가상 주소<br/>페이지#2, 오프셋500] --> PT[페이지 테이블]
    PT -->|프레임#7| CALC[물리 주소 계산]
    CALC --> PA[물리 주소<br/>프레임#7, 오프셋500]
    
    style PT fill:#ffe6cc
    style PA fill:#e6f2ff
```

### 페이지 크기의 트레이드오프

**큰 페이지**: 페이지 테이블 크기 감소, 내부 단편화 증가
**작은 페이지**: 페이지 테이블 크기 증가, 내부 단편화 감소
일반적으로 4KB가 균형점입니다.

### 단편화와의 관계

페이징은 **외부 단편화를 해결**합니다. 고정 크기이므로 어떤 프레임에든 배치 가능합니다. 하지만 **내부 [[메모리 단편화]]는 발생**합니다 (평균 0.5 페이지 손실).

### 백엔드 개발과의 연관성

리눅스에서 `getconf PAGESIZE` 명령으로 페이지 크기를 확인할 수 있습니다. 데이터베이스도 페이지 단위로 데이터를 관리하며, PostgreSQL은 8KB 페이지를 사용합니다.
