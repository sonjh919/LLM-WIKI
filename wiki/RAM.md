## 왜 이 이름인가

RAM = Random Access Memory (임의 접근 메모리). 데이터가 저장된 위치와 상관없이 어디든 동일한 속도로 접근할 수 있어서 "Random Access"라는 이름이 붙음. 반대 개념은 순차 접근(Sequential Access)인 테이프 저장장치.

## 기존 문제

CPU는 매우 빠르지만, [[Storage]]는 느리다. CPU가 매번 보조기억장치에서 데이터를 가져오면 대부분의 시간을 기다리는 데 쓰게 된다. "현재 실행 중인 프로그램과 데이터만 빠른 곳에 올려놓으면 되지 않을까?"

## 어떻게 해결하는가

RAM은 현재 실행 중인 프로그램과 데이터를 임시로 저장하는 **주기억장치**.

- **휘발성** — 전원이 꺼지면 데이터가 사라짐 (↔ Storage는 비휘발성)
- **임의 접근** — 어떤 주소든 동일한 속도로 접근 가능
- **CPU의 작업 공간** — CPU는 RAM에 올라온 데이터만 처리할 수 있음. [[System Bus]]를 통해 CPU와 데이터를 주고받음

## 백엔드 개발에서의 활용

- **JVM 메모리 구조**: Spring 앱 실행 시 JVM의 힙(객체), 스택(메서드 호출)이 RAM에 할당됨. `-Xmx` 옵션으로 힙 크기를 조절하는 것은 곧 RAM 사용량을 조절하는 것
- **OOM(Out of Memory)**: RAM 할당 한계를 넘기면 발생. 메모리 누수, 과도한 캐싱, 대용량 쿼리 결과를 한 번에 로딩할 때 주의
- **인메모리 캐시**: Redis, Caffeine 같은 캐시가 빠른 이유는 디스크가 아닌 RAM에 데이터를 저장하기 때문
- **동시 요청 처리**: 각 요청마다 스레드가 스택 메모리를 소비. 동시 접속자가 많을수록 RAM 소비가 늘어남
