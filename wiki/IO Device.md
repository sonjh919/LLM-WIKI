## 왜 이 이름인가

I/O Device = Input/Output Device (입출력 장치). 컴퓨터 내부(CPU, RAM)와 외부 세계 사이에서 데이터를 "들여오고(Input) 내보내는(Output)" 장치이기 때문.

## 기존 문제

CPU와 RAM만으로는 외부와 소통할 수 없다. 사용자 입력을 받을 수도, 결과를 보여줄 수도, 네트워크로 데이터를 주고받을 수도 없다. "컴퓨터 내부와 외부를 연결하는 통로가 필요하다."

## 어떻게 해결하는가

I/O 장치가 [[System Bus]]를 통해 CPU, RAM과 데이터를 주고받는다.

- **입력 장치** — 키보드, 마우스, 마이크
- **출력 장치** — 모니터, 프린터, 스피커
- **입출력 장치** — 네트워크 카드(NIC), USB, 터치스크린

### I/O가 느린 이유

I/O 장치는 CPU에 비해 극도로 느리다. CPU가 I/O 완료를 매번 기다리면 대부분의 시간을 낭비하게 됨 → 인터럽트, DMA 같은 메커니즘이 이를 해결.

## 백엔드 개발에서의 활용

- **네트워크 I/O**: 서버의 핵심 I/O 장치는 네트워크 카드. HTTP 요청/응답, DB 커넥션, 외부 API 호출이 전부 네트워크 I/O
- **I/O 바운드 서버**: 대부분의 백엔드 서버는 CPU 연산보다 I/O 대기(DB, 네트워크) 시간이 길다. 그래서 Spring WebFlux, 비동기 처리가 의미 있는 것
- **디스크 I/O**: 파일 업로드/다운로드, 로그 쓰기도 I/O. `java.nio`의 Non-blocking I/O가 이 맥락
- **I/O 멀티플렉싱**: 하나의 스레드가 여러 I/O 채널을 동시에 감시하는 기법. Linux의 epoll, Java NIO의 Selector가 이 원리
