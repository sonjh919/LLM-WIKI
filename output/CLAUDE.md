# output/ — 결과물

위키 지식을 기반으로 생성한 최종 결과물을 저장한다.

## 결과물 유형
- 시각화: canvas, 흐름도, 구조도, 마인드맵
- 학습 계획: 맞춤형 학습 로드맵
- 리마인드 노트: 빠른 복습용 요약

## 규칙
- 결과물은 wiki/ 페이지를 기반으로 생성한다
- 파일명에 유형을 명시한다 (예: `canvas_Spring_전체구조.canvas`, `plan_DB학습계획.md`)

## 시각화 적용 기준
- 시각화는 **공간적·물리적 관계**가 본질인 주제에만 생성한다 (컴퓨터 부품, 네트워크 토폴로지, 아키텍처 다이어그램 등)
- **개념적 연관**(코딩 원칙, 설계 패턴, 학습 분류 등)은 시각화하지 않는다 — Obsidian graph view가 wikilink로 자동 그래프를 제공하므로 수동 canvas는 중복
- 개념 원칙에 실전 가치를 주려면 조감도가 아닌 **체크리스트·결정 트리·표**를 검토한다
- 페이지가 쌓일 때 "조감이 필요한가?"를 먼저 묻기보다, graph view로 충분한지부터 확인한다
