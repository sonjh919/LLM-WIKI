# 학습 후보 백로그

학습 세션 중 "이것도 파보고 싶다"고 튀어나왔지만 그 세션에서는 소화하지 않은 후보들. 세션 종료 시점에 추가하고, 학습 우선순위 결정에 사용.

## 읽는 법

- **신호 강도**: 서로 다른 세션·페이지에서 등장한 횟수. ⭐ 1회, ⭐⭐ 2회, ⭐⭐⭐ 3회 이상
- **상태**:
  - `후보` — 아직 미학습
  - `진행중` — 다음 학습 대상으로 찜
  - `완료` — 독립 페이지로 승격됨 (역사 기록용으로 유지)

---

## 데이터베이스 ⭐⭐⭐ (최우선)

### DB 인덱스 내부 구조
- **맥락**: [[Storage]] 복습 — 풀 테이블 스캔 vs 인덱스로 I/O 절감
- **서브 질문**: B-Tree·B+Tree는 왜 DB 인덱스의 기본인가? 해시 인덱스는 언제? 복합 인덱스 순서는 왜 중요?
- **신호**: ⭐⭐ (Storage·RAM 세션)
- **상태**: 후보

### DB 버퍼 풀·캐시
- **맥락**: [[Storage]] 복습 — "같은 쿼리 두번째가 빠른 이유". [[RAM]]의 인메모리 캐시와 연결
- **서브 질문**: InnoDB Buffer Pool 동작, 히트율 지표, LRU 변형 알고리즘
- **신호**: ⭐⭐ (Storage·RAM 세션)
- **상태**: 후보

### 트랜잭션·ACID
- **맥락**: [[Storage]] 복습 중 "DB 더 파보고 싶다" 메타 제안
- **서브 질문**: 격리 수준, 락, WAL, MVCC
- **신호**: ⭐
- **상태**: 후보

### EXPLAIN과 쿼리 플래너
- **맥락**: [[Storage]] 복습 — "느린 쿼리 원인이 SQL인지 디스크 I/O인지"
- **서브 질문**: 실행 계획 읽는 법, 풀 스캔 식별, 옵티마이저 통계
- **신호**: ⭐
- **상태**: 후보

## 네트워크 / 비동기 ⭐⭐⭐

### 소켓 개념 (사용자 자기 인식)
- **맥락**: [[IO Device]] 복습 — "소켓의 정확한 개념을 이해하진 못한 것 같아"라고 본인 인식
- **서브 질문**: 소켓 = 무엇의 추상화? OS 파일 디스크립터와의 관계, TCP/UDP 소켓 차이
- **신호**: ⭐
- **상태**: 후보

### I/O 멀티플렉싱 상세 (epoll·Selector)
- **맥락**: [[IO Device]] 복습 — "이벤트 루프?" 추측까지만 감
- **서브 질문**: epoll의 LT/ET 모드, Selector 내부, select vs poll vs epoll
- **신호**: ⭐
- **상태**: 후보

### Netty
- **맥락**: [[DMA]]·[[IO Device]] — "왜 Netty가 빠른가"의 답을 이루는 기반 개념 정리됨. Netty 자체는 미학습
- **서브 질문**: 이벤트 루프 그룹, Channel·Pipeline, zero-copy 활용
- **신호**: ⭐⭐ (System Bus·IO Device 세션)
- **상태**: 후보

### Spring WebFlux / 리액티브
- **맥락**: [[IO Device]] 복습 — I/O 멀티플렉싱의 상위 추상
- **서브 질문**: Mono/Flux, 백프레셔, MVC 대비 실제 이득 조건
- **신호**: ⭐⭐ (System Bus·IO Device)
- **상태**: 후보

### zero-copy / sendfile
- **맥락**: [[DMA]] 페이지 — Kafka·Nginx 성능의 근거
- **서브 질문**: 복사 횟수 구체 경로, FileChannel.transferTo 내부
- **신호**: ⭐
- **상태**: 후보

- **신호**: ⭐
- **상태**: 후보
