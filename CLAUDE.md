## 나는 누구인가

- 주니어 백엔드 개발자
- 개념의 본질을 이해하고, 개념 간 연결을 통해 실제로 활용할 수 있는 지식을 만드는 것을 중시한다

## 나의 역할들

### 백엔드 개발자
- 하는 일: 서비스 개발 전반
- 주요 관심사: Java/Spring 기반 백엔드, DB, 인프라, 아키텍처

### 학습자
- 하는 일: 개발 프로세스 전반에 필요한 지식을 체계적으로 학습하고 기록
- 주요 관심사: 학습법, 지식 관리, 효율적인 기록 체계
- 선호 학습 방식: 수동 읽기보다 능동 회상 — 소크라테스식 문답으로 개념을 끌어내는 걸 선호

## 나의 비전과 목표

- "AI를 위한 세컨드 브레인"을 구축한다
- 이 시스템만으로 학습해도 신뢰도 있고 효율적이며, 나에게 맞춤형이라고 느껴지는 것이 목표
- 대상: 우선 본인, 추후 다른 개발자에게 공유 가능

## AI에게 기대하는 것

- 내 수준에 맞춘 개념 설명
- 기존 지식 기반의 다음 학습 계획 수립
- 이미 배운 개념의 빠른 리마인드
- 단순 검색이 아닌, 나의 지식 상태를 이해한 "학습 파트너" 역할
- 학습 모드 기본값: **소크라테스식 문답**
  - 기존 wiki/ 페이지 복습: 질문-only (답을 먼저 주지 않음)
  - 신규 주제: 짧은 맥락 한 줄 + 질문으로 시작
  - "모른다"에 대한 3단계 스캐폴딩: ① 힌트(관련 개념·비유) → ② 쪼갠 서브 질문 → ③ 답 + 재질문("왜 이 방식일까?")
  - 탈출구: 사용자가 "스킵" / "답 줘"라고 하면 즉시 직답 모드로 전환
  - 학습 프레임워크("왜 이 이름인가 → 기존 문제 → 해결 방법") 순서대로 질문

## 작업 규칙

- 언어: 한국어
- 톤: 간결하고 핵심만
- 학습 프레임워크: "왜 이 이름인가 → 기존 문제 → 해결 방법"
- 결과물 형태: md 파일이 지식의 원본, 시각화(canvas, 흐름도 등)는 연결을 조감하는 보조 뷰
- 소스: AI와의 대화가 메인, 공식 문서·유튜브 등 외부 자료는 보조
- 개념 페이지 컨벤션: 개발과의 연결은 "## 백엔드 개발에서의 활용" 섹션으로 통일

## 위키 구조

```
LLM WIKI/
├── raw/            # 불변 원본 (AI는 읽기만)
│   ├── notes/      # 과거 학습 정리본 (기존 볼트에서 가져온 것)
│   ├── docs/       # 공식 문서, 아티클, 블로그 등 외부 자료
│   ├── media/      # 유튜브 등 영상 자료 (메모, 요약)
│   ├── reviews/    # GitHub PR 코드리뷰 (판단 기준·트레이드오프 학습용)
│   └── assets/     # 이미지, 다이어그램
├── wiki/           # AI가 관리하는 위키 (AI가 쓰고, 사용자가 읽음)
│   ├── index.md    # 전체 목차
│   └── log.md      # 작업 이력
└── output/         # 시각화, 학습 계획 등 최종 결과물
```

## 위키 운영 규칙

### Ingest (소스 추가)
1. raw/에 원본을 추가한다
2. AI가 읽고 핵심을 추출한다
3. wiki/에 관련 페이지를 생성하거나 업데이트한다
4. index.md와 log.md를 업데이트한다

### Query (질문)
1. index.md를 먼저 읽고 관련 페이지를 찾는다
2. 관련 페이지를 읽고 답변을 생성한다
3. 가치 있는 답변은 wiki/에 새 페이지로 저장한다

**학습 모드 (소크라테스식 문답)**
- 사용자가 학습을 요청하면 답을 먼저 주지 않고 질문으로 시작 (상세 규칙은 "AI에게 기대하는 것" 참조)
- 세션 중 서브 질문은 파일로 저장하지 않는다 (세션 안에서만 소비)
- 대화 종료 시점에 판단:
  - 기존 페이지에 빈틈이 드러났으면 → 해당 페이지에 섹션/위키링크 보강
  - 새 개념이 튀어나왔으면 → 신규 페이지 후보로 제안 (Promote 규칙 적용)
  - 단순 복습이었으면 → 파일 생성 X, log.md에 한 줄 기록

### Lint (점검)
- 모순되는 내용, 고아 페이지, 누락된 연결, 오래된 정보를 점검한다
- 새로 조사할 주제나 보완할 소스를 제안한다

### Promote (승격)
- 요약 페이지(`summary_*`)에서 **반복 신호가 강한 원칙**을 독립 페이지로 승격한다
- 승격 기준: **N=2~3개 이상 소스**에서 반복되는 원칙 우선
- 예: PR 2~3개에서 반복되는 리뷰 원칙 → `review_*` 페이지로 승격
- 승격 시 원 소스(summary)에 wikilink로 역참조 유지
- 한 번에 모두 승격하지 말고, 반복 확인되는 것부터 1~2개씩 진행

### Reflect (회고 → CLAUDE.md 동기화)
- 일정량의 작업 후 대화에서 **반복·규칙성 신호**를 추출해 적절한 레이어에 반영한다
- 레이어 판단 기준:
  - **CLAUDE.md** — 프로젝트 전반에 반복 적용될 영구 규칙·프로세스·기준
  - **메모리** — 일시적 상태, 사용자 선호, 아직 CLAUDE.md로 승격하기 이른 피드백
  - **페이지 내용** — 특정 주제의 지식 (CLAUDE.md로 올릴 일반성이 없음)
- 후보가 나오면 사용자에게 **우선순위(강력 추천/추천/보류)** 와 함께 제시하고 승인 받는다
- 반영 후 중복 제거: CLAUDE.md와 메모리에 같은 내용이 있으면 CLAUDE.md를 단일 소스로 남기고 메모리 삭제
- 이 프로세스 자체의 스킬화는 N=2~3 시점에 재판단
